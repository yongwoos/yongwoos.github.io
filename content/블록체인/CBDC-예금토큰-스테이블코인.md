## 예금 토큰(Deposit Token)

시중 은행이 예금시 코인 발급해주는 것.

사실상 화폐기능

**은행에 돈 맡기고 → 그 예금을 토큰화한 것**

---

## 구조

```
내가 은행에 100만원 입금
       ↓
은행이 "100만원 예금 토큰" 발행해서 내 지갑에 줌
       ↓
토큰 = 예금 청구권 (언제든 100만원 돌려받을 수 있는 증서)
```

네가 말한 거 그대로야. 돈 맡기고 토큰 받는 것.

---

## 그럼 스테이블코인이랑 뭐가 달라?

헷갈리는 부분이 여기야.

| | 예금 토큰 | 스테이블코인 (USDC 등) |
|--|----------|----------------------|
| 발행 주체 | 은행 (규제 기관) | 민간 기업 |
| 법적 지위 | 예금 그 자체 | 예금 아님 |
| 예금자 보호 | 적용됨 | 적용 안 됨 |
| 규제 | 은행법 적용 | 불명확 |
| 신뢰 기반 | 국가·중앙은행 | 발행사 신뢰 |

USDC는 Circle이라는 회사가 발행 → Circle 망하면 위험

예금 토큰은 KB국민은행 같은 규제 은행이 발행 → 예금자보호법 적용

---

## CBDC랑도 달라

```
CBDC        = 중앙은행이 직접 발행 (한국은행 → 디지털 원화)
예금 토큰   = 시중은행이 발행 (국민은행 → 예금 토큰)
스테이블코인 = 민간 기업이 발행 (Circle → USDC)
```

```
신뢰도 순서:
CBDC > 예금 토큰 > 스테이블코인
```

---

## 왜 만드냐?

예금을 토큰화하면:

```
기존 송금:
A은행 → 한국은행 → B은행 (중간 정산, 1~2일)

예금 토큰:
A의 토큰 → 스마트 컨트랙트 → B에게 즉시 전달
(수수료 없음, 24시간, 즉시)
```

특히 **기업 간 결제, 무역금융**에서 효율이 극적으로 올라가.

---

## 현실에서는

아직 대부분 실험·파일럿 단계야.

- JP모건 → JPM Coin (기관 간 결제용)
- 한국 → 한국은행 CBDC 파일럿 진행 중
- BIS → 여러 나라 중앙은행과 예금 토큰 실험

**결론:** 네 직관 맞아. 돈 맡기고 토큰 받는 건데, 그 토큰이 법적으로 예금이랑 동일한 지위를 가진다는 게 핵심이야.

## CDBC
중앙은행이 은행간 거래시 발행하는 코인

## 스테이블코인
민간이 발행하는 화폐나 채권에 1:1 페깅된 코인

## 비교

```
CBDC         = 중앙은행 발행 디지털 통화
예금 토큰    = 시중은행 발행 디지털 통화
스테이블코인 = 민간 발행 디지털 통화

셋 다 하는 일: 가치 저장 + 결제 + 송금
```

기능만 보면 거의 동일해. 네 말이 맞아.

---

## 차이는 기능이 아니라 **발행 구조**

```
CBDC
중앙은행 → 직접 발행
→ 국가 부채 (=법정화폐 그 자체)

예금 토큰
시중은행 → 발행
→ 은행 부채 (예금이 원래 은행 부채임)
→ 중앙은행이 뒤에서 보증

스테이블코인
민간기업 → 발행
→ 기업 부채
→ 국가 보증 없음
```

---

## 왜 CBDC 놔두고 예금 토큰을 따로 만드냐?

여기가 핵심이야.

**CBDC를 일반인한테 직접 뿌리면 생기는 문제:**

```
국민들이 시중은행 예금 → CBDC로 대거 이동
       ↓
시중은행 예금 급감
       ↓
은행 대출 재원 없어짐
       ↓
신용창조 붕괴 → 경제 위기
```

그래서 중앙은행 입장에서:

```
CBDC는 은행 간 결제용으로만 쓰고
일반인 대상은 예금 토큰으로 대신하자
→ 기존 은행 시스템 유지하면서
→ 블록체인 효율만 가져오자
```

---

## 결국 세 줄 요약

> 기능은 같다. 차이는 누가 발행하고 누가 책임지냐.
> CBDC는 은행 시스템 흔들까봐 소매용으로 못 풀고 있음.
> 예금 토큰은 그 공백을 시중은행이 채우는 타협안.
