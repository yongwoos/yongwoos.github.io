---
title: 우분투 앱 삭제
---
Ubuntu 대시보드(프로그램 표시)에는 아이콘이 보이지만, 우분투 소프트웨어(앱스토어) 목록에는 나타나지 않는 앱을 삭제하는 방법은 크게 두 가지입니다. 보통 터미널을 이용하거나, 바로가기 파일(`.desktop`)을 직접 제거하는 방식으로 해결할 수 있습니다.

---

### 1. 터미널을 이용한 패키지 삭제
앱스토어에 뜨지 않는다면 일반적인 `snap` 패키지가 아닌 `apt` 패키지로 설치되었을 확률이 높습니다.

* **앱 이름 찾기:** 정확한 패키지 명을 모를 경우 아래 명령어로 검색합니다.
    `dpkg -l | grep '앱이름'`
* **삭제하기:** 이름이 확인되었다면 아래 명령어를 입력하세요.
    ```bash
    sudo apt purge 패키지명
    sudo apt autoremove
    ```
    *(참고: `purge`는 설정 파일까지 깔끔하게 지워줍니다.)*

---

### 2. `.desktop` 파일 직접 삭제 (아이콘만 남은 경우)
앱은 이미 지워졌는데 대시보드에 **아이콘 잔상**만 남은 경우입니다. 우분투는 `/usr/share/applications` 또는 `~/.local/share/applications` 경로에 있는 `.desktop` 파일을 읽어 대시보드에 표시합니다.

1.  **경로 이동:** 터미널에서 아래 경로로 들어갑니다.
    * 시스템 전체 앱: `/usr/share/applications`
    * 사용자 설치 앱: `~/.local/share/applications`
2.  **파일 확인:** 해당 폴더에서 삭제하고 싶은 앱 이름과 유사한 `.desktop` 파일을 찾습니다.
3.  **삭제:** ```bash
    # 시스템 경로인 경우 sudo 권한 필요
    sudo rm /usr/share/applications/앱이름.desktop
    ```
4.  **반영:** 파일을 지우면 대시보드에서 즉시 혹은 재로그인 시 아이콘이 사라집니다.

---

### 3. Snap 앱인 경우 확인
혹시 모르니 Snap으로 설치되었는지도 확인해 보세요.
```bash
snap list
# 목록에 있다면
sudo snap remove 앱이름
```

---

### 요약 가이드
| 상황 | 해결 방법 |
| :--- | :--- |
| **명령어로 설치한 앱** | `sudo apt purge 패키지명` |
| **앱은 지웠는데 아이콘만 남음** | `.local/share/applications` 에서 `.desktop` 파일 삭제 |
| **압축 풀어서 실행만 했던 앱** | 설치 폴더 자체를 삭제하고 `.desktop` 파일 제거 |
