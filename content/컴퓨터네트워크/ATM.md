## 1. ATM의 개요
* **정의:** 정보를 일정한 크기의 **셀(Cell)** 단위로 나누어 전송하는 통신 방식입니다.
* **배경:** 광대역 종합정보통신망(B-ISDN)을 구현하기 위한 핵심 전송 기술로 선택되었습니다.
* **전송 방식:** 회선 교환의 실시간성과 패킷 교환의 유연성을 결합하였으며, 통계적 시분할 다중화(Statistical TDM) 방식을 사용합니다.

---

## 2. ATM 셀(Cell)의 구조
ATM의 가장 큰 특징은 데이터 단위가 **53바이트**로 고정되어 있다는 점입니다.

* **전체 크기:** 53바이트
* **헤더(Header):** 5바이트
* **페이로드(Payload):** 48바이트 (실제 사용자 데이터)



---

## 3. 헤더(Header) 주요 필드별 역할
시험 문제에서 각 필드의 명칭과 기능을 묻는 경우가 많으므로 반드시 숙지해야 합니다.

| 필드명 | 풀네임 | 주요 기능 |
| :--- | :--- | :--- |
| **VPI** | Virtual Path Identifier | 가상 경로 식별자 |
| **VCI** | Virtual Channel Identifier | 가상 채널 식별자 |
| **PT** | Payload Type | 페이로드에 실린 정보의 종류(사용자 데이터, 망 관리 등) 표시 |
| **CLP** | Cell Loss Priority | 망 혼잡 시 셀의 폐기 우선순위 결정 (1일 경우 우선 폐기) |
| **HEC** | Header Error Control | 헤더 오류 검사 및 교정 (**CRC** 방식 사용) |

> **⚠️ 주의:** HEC는 **해밍코드(Hamming Code)** 가 아니라 **CRC(순환 중복 검사)** 방식을 사용한다는 점이 오답으로 자주 출제됩니다!

---

## 4. ATM의 주요 특징
1. **고정 길이 셀:** 데이터 단위가 일정하여 하드웨어적으로 빠른 처리가 가능하고 지연 시간이 적습니다.
2. **연결 지향성:** 데이터 전송 전 가상 회로(Virtual Circuit)를 먼저 설정하는 연결 지향성 프로토콜입니다.
3. **QoS 보장:** 서비스 종류별로 서로 다른 품질(QoS)을 제공할 수 있어 음성, 영상 등 실시간 데이터 처리에 유리합니다.
4. **비동기식 전송:** 대역폭을 미리 할당하지 않고 데이터가 발생할 때만 셀을 전송하므로 효율적입니다.

### 💡 시험 대비 팁
* ATM 셀의 총 크기(**53바이트**)와 헤더 크기(**5바이트**)를 정확히 외우세요.
* HEC의 오류 제어 방식은 **CRC**라는 점을 잊지 마세요.
* B-ISDN의 근간이 되는 기술이라는 점도 함께 기억해 두시면 좋습니다.
