여러 VLAN을 하나의 링크로 통신하게 해주는 기술

**Trunk**는 **여러 VLAN의 데이터를 하나의 링크로 동시에 보내는 방식**입니다.

왜 필요한지부터 보겠습니다.

스위치 A와 B가 있고 VLAN이 2개라고 가정합니다.

```text
스위치 A                 스위치 B

VLAN10  ───────────── VLAN10
VLAN20  ───────────── VLAN20
```

이렇게 연결하면 VLAN마다 케이블이 하나씩 필요합니다.

VLAN이 100개면?

**케이블 100개가 필요**합니다. 비효율적이죠.

그래서 **Trunk**를 사용합니다.

```text
스위치 A
     │
  Trunk 링크
     │
스위치 B
```

케이블 **1개**로

* VLAN10
* VLAN20
* VLAN30
* ...

을 모두 보냅니다.

그럼 스위치 B는 어떻게 "이 프레임이 VLAN10인지 VLAN20인지" 알까요?

그래서 **IEEE 802.1Q 태그(Tag)**를 붙입니다.

예를 들어

```text
원래 Ethernet Frame

[MAC][MAC][DATA][FCS]
```

Trunk에서는

```text
[MAC][MAC][802.1Q Tag][DATA][FCS]
```

가 됩니다.

802.1Q 태그 안에

> **VLAN ID = 10**

이 들어있으면 VLAN10으로 처리하고,

> **VLAN ID = 20**

이면 VLAN20으로 처리합니다.

예를 들면

```text
PC1
 ↓
VLAN10
 ↓
스위치 A
 ↓
Trunk
(태그: VLAN10)
 ↓
스위치 B
 ↓
VLAN10 포트
 ↓
PC2
```

### Access Port와 비교

**Access Port**

* PC 연결
* VLAN 하나만 전달
* 태그 없음

```text
PC
 │
Access Port
 │
VLAN10
```

**Trunk Port**

* 스위치 ↔ 스위치
* 스위치 ↔ L3 스위치
* 스위치 ↔ 라우터

여러 VLAN을 동시에 전달합니다.

```text
VLAN10
VLAN20
VLAN30
   │
 Trunk
   │
VLAN10
VLAN20
VLAN30
```

한 줄로 기억하면 됩니다.

* **Access = VLAN 하나**
* **Trunk = VLAN 여러 개(802.1Q 태그 사용)**

# VLAN
VLAN(Virtual LAN)은 **물리적으로는 하나의 스위치(LAN)지만, 논리적으로 여러 개의 LAN으로 나누는 기술**입니다.

핵심은 **"스위치 하나를 여러 개의 독립된 스위치처럼 사용하는 것"**입니다.

예를 들어 회사에 스위치 하나가 있습니다.

```text
스위치

A부서 PC
B부서 PC
C부서 PC
```

그냥 연결하면 모두 같은 LAN이므로 브로드캐스트를 서로 받고 통신도 가능합니다.

하지만 VLAN을 설정하면

```text
VLAN10 (A부서)
A1
A2

VLAN20 (B부서)
B1
B2

VLAN30 (C부서)
C1
C2
```

처럼 **하나의 스위치 안에 3개의 독립된 LAN**이 생깁니다.

그러면

* VLAN10 ↔ VLAN10 : 통신 가능
* VLAN10 ↔ VLAN20 : 통신 불가
* VLAN20 ↔ VLAN30 : 통신 불가

입니다.

### 왜 사용하나?

1. **보안**

인사팀과 개발팀을 분리할 수 있습니다.

```text
인사팀 VLAN
↓

개발팀 VLAN

직접 통신 불가
```

2. **브로드캐스트 감소**

브로드캐스트가 VLAN 내부에서만 퍼집니다.

```text
VLAN10
브로드캐스트
↓
VLAN10 안에서만 전달
```

다른 VLAN에는 가지 않습니다.

3. **관리 편리**

부서별, 층별, 프로젝트별로 논리적으로 쉽게 분리할 수 있습니다.

### 다른 VLAN끼리는?

스위치만으로는 **다른 VLAN끼리 통신할 수 없습니다.**

반드시 **라우터**나 **L3 스위치**가 필요합니다.

이를 **Inter-VLAN Routing**이라고 합니다.

```text
VLAN10
   │
   │
L3 스위치(또는 라우터)
   │
   │
VLAN20
```

### Static VLAN과 Dynamic VLAN

**① Static VLAN(포트 기반 VLAN)**

가장 많이 사용합니다.

```text
포트1 → VLAN10
포트2 → VLAN20
포트3 → VLAN10
```

포트에 VLAN을 직접 지정합니다.

PC를 다른 포트에 꽂으면 그 포트의 VLAN을 따릅니다.

**② Dynamic VLAN(MAC 기반 VLAN)**

```text
MAC 주소 확인
↓
DB 조회
↓
VLAN 자동 지정
```

어느 포트에 꽂아도 **MAC 주소를 보고 VLAN을 자동으로 결정**합니다.

### Trunk와 Access

VLAN 공부하면 반드시 같이 나오는 개념입니다.

**Access Port**

한 개의 VLAN만 전달합니다.

```text
PC
 │
Access Port
 │
VLAN10
```

보통 PC가 연결됩니다.

**Trunk Port**

여러 VLAN을 동시에 전달합니다.

```text
스위치
   │
Trunk
   │
스위치
```

VLAN10, VLAN20, VLAN30의 프레임을 모두 전달하기 위해 **IEEE 802.1Q 태그**를 붙여 어떤 VLAN의 프레임인지 표시합니다.

### 한 줄로 정리

* **VLAN = 하나의 물리적 스위치를 여러 개의 논리적 LAN으로 분리하는 기술**
* **Static VLAN = 포트 기준**
* **Dynamic VLAN = MAC 주소 기준**
* **같은 VLAN끼리만 스위치만으로 통신 가능**
* **다른 VLAN끼리는 라우터나 L3 스위치가 필요**
* **스위치 간 여러 VLAN을 전달할 때는 Trunk(802.1Q)를 사용**
