---
title: 트랜스폰더
---
**Trans**mitter + res**ponder** 의 합성어(Transponder)로, **신호를 수신하면 자동으로 응답 신호를 송신하는 장치**입니다.

### 기본 동작 원리

```
질문 신호 수신 → 신호 처리 → 응답 신호 송신
  (interrogation)              (reply)
```

수신 주파수와 송신 주파수가 **다른 것**이 핵심 특징입니다. (같으면 간섭 발생)


### 주요 사용 분야

**① 항공 (ATC Transponder)**
- 레이더 지상국이 질문 신호 송신 → 항공기 트랜스폰더가 자동 응답
- 응답에 항공기 식별코드(Squawk), 고도 정보 포함
- Mode A / Mode C / Mode S 방식

**② 위성통신**
- 위성에 탑재된 트랜스폰더가 지상국 신호를 받아 주파수 변환 후 재송신
- 예: 상향링크 14GHz → 하향링크 12GHz (Ku밴드)
- 위성의 핵심 중계 장치

**③ RFID**
- 리더기 신호 → 태그(수동 트랜스폰더) 응답
- 교통카드, 물류 태그 등

**④ 해상 (AIS/SART)**
- 선박 식별 및 조난 위치 발신

| 항목 | 내용 |
|------|------|
| 수/송신 주파수 | 반드시 다름 (이중화, duplex) |
| 동작 방식 | 수동적(passive) — 질문 받아야 응답 |
| 위성 트랜스폰더 | 주파수 변환 + 증폭 후 재송신 |
| 대역폭 | 트랜스폰더 1개당 할당 대역폭 존재 (예: 36MHz/72MHz) |

위성 트랜스폰더는 위성에서 신호를 수신하여 주파수를 변환하고 증폭한 뒤 다시 지상으로 송신하는 역할

단일주파수변환(Single Frequency Conversion) 시스템: 수신 주파수를 중간 주파수 과정 없이 한 번에 송신 주파수로 변환하는 방식입니다. 

이중주파수변환(Double Frequency Conversion) 시스템: 중간 주파수(IF) 단계를 거쳐 두 번에 걸쳐 주파수를 변환하는 방식으로, 필터 설계가 용이하고 안정적입니다. 

기저주파수검파(Regenerative) 시스템: 수신된 신호를 기저대역으로 복조하여 재생한 후 다시 변조하여 송신하는 방식으로, 신호의 품질을 높일 수 있습니다.

## 위성 트랜스폰더 (Satellite Transponder)

### 위성 통신 기본 구조

```
지상국 A                위성                  지상국 B
  │                      │                          │
  │──── 상향링크 ────────▶│             │
  │     (Uplink)         │ [트랜스폰더]              │
  │                      │ 수신→변환→증폭→송신     │
  │                      │──── 하향링크 ────────▶│
  │                      │     (Downlink)            │
```

위성은 **중계기** 역할 — 스스로 신호를 만들지 않고, 받아서 변환 후 재송신

---

### 트랜스폰더 내부 동작

```
수신 안테나
    │
    ▼
[LNA] ← 저잡음 증폭기 (Low Noise Amplifier)
    │        신호가 매우 약하므로 먼저 증폭
    ▼
[주파수 변환기] ← 상향링크 주파수 → 하향링크 주파수로 변환
    │              (간섭 방지 + 대기 투과 특성 최적화)
    ▼
[HPA] ← 고출력 증폭기 (High Power Amplifier)
    │        지상까지 도달할 만큼 강하게 증폭
    ▼
송신 안테나
```

---

### 주파수 대역별 정리

| 대역 | 상향링크 | 하향링크 | 특징 |
|------|---------|---------|------|
| C밴드 | 6 GHz | 4 GHz | 강우 영향 적음, 안테나 큼 |
| Ku밴드 | 14 GHz | 12 GHz | 소형 안테나 가능, 강우 감쇠 있음 |
| Ka밴드 | 30 GHz | 20 GHz | 광대역, 강우 감쇠 큼 |

> 상향 > 하향 주파수인 이유: 높은 주파수일수록 감쇠 크므로, **출력이 강한 지상국이 높은 주파수로 올려 보내고**, 출력이 약한 위성은 낮은 주파수로 내려보냄

---

### 트랜스폰더 대역폭과 다중화

트랜스폰더 1개 = 할당된 주파수 슬롯 (예: **36MHz** 또는 **72MHz**)

위성 하나에 트랜스폰더 여러 개 탑재 → 각각 다른 채널 처리

**다중화 방식:**
- **FDMA** — 주파수 분할, 트랜스폰더 대역폭을 여러 사용자가 나눔
- **TDMA** — 시간 분할
- **CDMA** — 코드 분할

---

### 트랜스폰더 종류

| 종류 | 설명 |
|------|------|
| **Bent-pipe** (투명형) | 신호를 그냥 변환·증폭만 함. 내용 처리 없음. 단순하고 신뢰성 높음 |
| **재생형 (Regenerative)** | 신호를 복조 → 재변조. 잡음 제거 가능. 복잡하지만 품질 좋음 |

시험엔 **Bent-pipe = 투명 중계기** 로 자주 출제됩니다.

---

### 시험 핵심 요약

- 트랜스폰더 = **LNA → 주파수변환 → HPA** 3단계
- 상향/하향 주파수 **반드시 다름** (같은 대역 내에서도 분리)
- 대역폭(예: 36MHz)이 트랜스폰더 용량 결정
- Bent-pipe vs 재생형 차이 기억
