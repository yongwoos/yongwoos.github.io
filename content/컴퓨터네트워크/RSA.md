- **공개키**: (e, n) — e는 암호화 지수, n은 모듈러스
- **개인키**: (d, n) — d는 복호화 지수, n은 모듈러스

**n은 둘 다 알고 있는 공유 숫자** e는 공개키.

---

## 암호화/복호화 공식

```
암호화: C = M^e mod n
복호화: M = C^d mod n
```
- `M ^ e % n`

지수가 e/d이고, mod가 n이야.

---

## 원리

n = p × q (두 소수의 곱)로 만들고, e와 d를 이렇게 설정해:

```
e × d ≡ 1 (mod φ(n))
```

이 관계 때문에 `(M^e)^d mod n = M`이 성립해. 오일러 정리 덕분이야.

응 정확해.

풀어쓰면:

```
암호화: C = M^e mod n
복호화: C^d mod n = M

즉: (M^e mod n)^d mod n = M
```

이게 성립하는 이유는 모듈러 산술 성질 때문에

```
(M^e mod n)^d mod n = M^(e×d) mod n
```

으로 바꿀 수 있고, e×d ≡ 1 (mod φ(n)) 으로 설계했기 때문에

```
M^(e×d) mod n = M^1 mod n = M
```

이 되는 거야.

핵심은 **e와 d가 서로 역원 관계**라는 거고, 그걸 φ(n) 기반으로 설계하는 게 RSA의 핵심 수학이야.
