| 프로토콜                        | 현재 사용       |
| --------------------------- | ----------- |
| IPv4                        | 매우 많이 사용    |
| IPv6                        | 매우 많이 사용    |
| IPX                         | 거의 사장됨      |
| AppleTalk (DDP)             | 사장됨         |
| X.25                        | 거의 사장됨      |
| CLNP (OSI Network Protocol) | 일부 특수망에서 사용 |

### 1. IPX

Novell NetWare에서 사용하던 네트워크 계층 프로토콜입니다.

```text
Ethernet
   ↓
IPX
   ↓
SPX(전송계층)
```

여기서 **SPX는 TCP 역할**, **IPX는 IP 역할**을 했습니다.

### 2. AppleTalk

애플 컴퓨터들이 예전에 사용하던 프로토콜입니다.

```text
Ethernet
   ↓
DDP (AppleTalk의 네트워크 계층)
   ↓
ATP
```

역시 IP 없이 자체적으로 주소 지정과 라우팅을 했습니다.

### 3. X.25

초기의 공중 데이터망에서 사용되었습니다.

은행망, 공공망 등에서 많이 쓰였지만 지금은 거의 사라졌습니다.

### 4. CLNP

OSI 7계층 모델에서 정의한 **IP의 경쟁 프로토콜**입니다.

```text
OSI
Network Layer
↓
CLNP
```

인터넷에서는 TCP/IP가 승리하면서 거의 쓰이지 않게 되었습니다.

### 왜 지금은 거의 IP뿐이냐?

1980~1990년대에는

* TCP/IP
* IPX/SPX
* AppleTalk
* DECnet
* X.25
* CLNP

등 여러 네트워크 계층 프로토콜이 경쟁했습니다.

하지만 인터넷이 TCP/IP를 표준으로 채택하면서 대부분 사라졌고, 현재는 사실상

* **IPv4**
* **IPv6**

두 가지가 인터넷의 표준 네트워크 계층 프로토콜입니다.

즉, **IP는 유일한 3계층 프로토콜이 아니라 현재 살아남은 사실상의 표준**이라고 이해하면 됩니다.


## IP 관련 프로토콜

| 프로토콜           | 역할                |
| -------------- | ----------------- |
| IP             | 패킷 전달, 주소 지정      |
| ICMP           | 오류 보고, 진단(ping 등) |
| IGMP           | IPv4 멀티캐스트 그룹 관리  |
| OSPF           | 라우터 간 경로 정보 교환    |
| RIP            | 라우팅 프로토콜          |
| IS-IS          | 대규모 네트워크 라우팅      |
| IPsec(AH, ESP) | IP 계층 보안          |

여기서 헷갈리는 점이 하나 있습니다.

### IP와 같은 '데이터 전달 프로토콜'

* IP
* (과거) IPX
* (과거) AppleTalk

이들은 **실제 패킷을 목적지까지 전달하는 네트워크 계층 프로토콜**입니다.

### IP를 보조하는 네트워크 계층 프로토콜

* ICMP
* IGMP
* OSPF
* RIP
* IS-IS

이들은 **IP를 대체하는 것이 아니라 IP 네트워크를 운영하기 위한 프로토콜**입니다.

예를 들어

* **IP**: 데이터를 목적지로 보낸다.
* **ICMP**: 목적지에 도달하지 못했다고 알려준다.
* **OSPF**: 어느 경로로 보내야 하는지 라우터끼리 정보를 교환한다.
* **IGMP**: 멀티캐스트 그룹 가입을 관리한다.

즉, 인터넷에서는

```text
IP  ← 핵심 데이터 전달
├─ ICMP (오류·진단)
├─ IGMP (멀티캐스트)
├─ OSPF (라우팅)
└─ IPsec (보안)
```

처럼 **IP를 중심으로 여러 3계층 프로토콜이 함께 동작**한다고 이해하면 됩니다.

여기서 **TCP는 4계층 프로토콜**이 맞습니다.

그런데 **ICMP와 IGMP는 예외**입니다.

```
IP Header
Protocol = 1
↓
ICMP
```

여기서 ICMP는 **TCP나 UDP 위에 있는 것이 아니라 IP 바로 위**에 있습니다. 그래서 **4계층이 아니라 3계층 프로토콜**로 분류합니다.

마찬가지로

```
IP Header
Protocol = 2
↓
IGMP
```

도 IP 바로 위에서 동작하므로 3계층입니다.

OSPF도 동일합니다.

```
IP Header
Protocol = 89
↓
OSPF
```

OSPF 역시 TCP나 UDP를 거치지 않습니다.

즉 **IP 헤더의 Protocol 필드는 "다음에 오는 상위 프로토콜"을 나타내는 필드**일 뿐, **그 프로토콜이 반드시 4계층이라는 뜻은 아닙니다.**

정리하면

* IP → TCP → 응용계층 (TCP는 4계층)
* IP → UDP → 응용계층 (UDP는 4계층)
* IP → ICMP (3계층)
* IP → IGMP (3계층)
* IP → OSPF (3계층)

ICMP, IGMP, OSPF는 **IP를 이용해 전달되지만 TCP/UDP를 사용하지 않는 네트워크 계층 프로토콜**이라서 3계층으로 분류됩니다. 이 점 때문에 처음 배우면 가장 많이 헷갈리는 부분입니다.
