## 1. IPv4 구조

$$
\text{IP} = \text{네트워크 부분} + \text{호스트 부분}
$$

* 네트워크 부분 → “어느 서브넷인지”
* 호스트 부분 → “그 안의 장비”


## 2. 클래스 A / B / C 구조

클래스는 “고정된 비트 분할 규칙”

| 클래스 | 구조               | 기본 마스크  | 의미  |
| --- | ---------------- | ------- | --- |
| A   | 네트워크 8 / 호스트 24  | $$/8$$  | 초대형 |
| B   | 네트워크 16 / 호스트 16 | $$/16$$ | 중형  |
| C   | 네트워크 24 / 호스트 8  | $$/24$$ | 소형  |


* A → 한 조직이 통째로 씀
* B → 중간 규모
* C → 작은 네트워크 다수 생성용

## 3. 클래스 범위

* A: 0.0.0.0 ~ 127.255.255.255 (1 ~ 126)
* B: 128.0.0.0 ~ 191.255.255.255 (128 ~ 191)
* C: 192.0.0.0 ~ 223.255.255.255 (192 ~ 223)

첫 옥텟 기준:

| 첫 옥텟 | 클래스/용도 |
|---------|------------|
| **0** | 예약 (현재 네트워크 지칭, 라우팅 불가) |
| 1~126 | **Class A** |
| **127** | 루프백 (127.0.0.0 ~ 127.255.255.255) |
| 128~191 | **Class B** |
| 192~223 | **Class C** |
| 224~239 | **멀티캐스트 (Class D)** |
| 240~255 | **예약 (Class E)** |


- **0.0.0.0~0.255.255.255** → 루프백이 아니라 **예약 대역** (현재 호스트/네트워크 의미)
- 루프백은 오직 **127.x.x.x** 하나뿐
- **240~255 (Class E)** — 예약/실험용, 시험에 종종 출제됨


### 암기 팁

```
0        → 예약
1~126    → A (0 빠짐 주의!)
127      → 루프백
128~191  → B
192~223  → C
224~239  → D (멀티캐스트)
240~255  → E (예약)
```
## 4. 왜 클래스가 사라졌는가

고정 구조 문제:

* C는 부족 (254개)
* B는 과다 (65534개 낭비)

👉 해결: CIDR


## 5. 호스트 주소의 의미

서브넷 내부 구조:

$$
\text{네트워크 주소} + \text{호스트 주소} + \text{브로드캐스트 주소}
$$


## 역할

### 네트워크 주소

* 서브넷 자체 식별자
* 사용 불가

### 호스트 주소

* 실제 장비 IP
* 우리가 PC/서버에 할당하는 값

### 브로드캐스트 주소

* 전체 전송용
* 사용 불가


## 6. 사용 가능한 호스트 수 공식

호스트 비트가 (h)개일 때:

$$
2^h - 2
$$

* -2 이유

  * 네트워크 주소 1개
  * 브로드캐스트 1개

## 7. 예시

$$
/29 \Rightarrow h = 3
$$

전체:

$$
2^3 = 8
$$

사용 가능:

$$
8 - 2 = 6
$$

구조:

* 네트워크 1개
* 호스트 6개
* 브로드캐스트 1개

## 8. CIDR 핵심

CIDR은 클래스의 고정 규칙을 깨는 방식

* /8

* /16

* /24
  → 과거 클래스 방식

* /20 /27 /29 /30
  → 현대 방식 (유연하게 분할)


## 9. 핵심 요약

* IP = 네트워크 + 호스트
* 클래스 = 옛날 고정 분할 (/8 /16 /24)
* 실제 장비 IP = 호스트 주소
* 서브넷에는 무조건 2개 예약 주소 존재
* 그래서 %%2^h - 2%%
* 현재 표준은 CIDR
