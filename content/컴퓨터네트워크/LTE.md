## 왜 상향/하향에 다른 방식을 쓰는가?

핵심 이유는 딱 하나: **단말기(스마트폰)의 배터리와 출력 한계**

---

### OFDM vs SC-FDMA 비교

| | 하향 (기지국→단말) | 상향 (단말→기지국) |
|--|--|--|
| 방식 | **OFDM** | **SC-FDMA** |
| 송신자 | 기지국 | 스마트폰 |
| PAPR | 높음 (문제 없음) | **낮음** (핵심 장점) |
| 주파수 효율 | 높음 | 동등 |
| 복잡도 | 수신단 단순 | 수신단 약간 복잡 |

---

### PAPR이 뭐고 왜 중요한가?

**PAPR = Peak-to-Average Power Ratio (최대전력 대 평균전력 비)**

OFDM은 수십~수백 개의 부반송파(subcarrier)를 **동시에** 송신합니다.

```
부반송파들이 우연히 같은 위상으로 겹치면:

↑ 합산 신호
│        ▲ 순간 피크
│       ╱╲
│      ╱  ╲        ← 평소엔 낮은 평균 출력
│─────╱────╲──────
│              ╲╱
```

피크 순간에 증폭기(PA)가 그 출력을 감당해야 함
→ 기지국은 전원 충분하므로 **괜찮음**
→ 스마트폰은 배터리 한계, 발열, 소형 PA → **PAPR 높으면 치명적**

---

### SC-FDMA의 해결법

OFDM에서 **DFT(이산 푸리에 변환) 전처리**를 한 단계 추가

```
[송신 데이터]
     │
  [DFT] ← 여기가 핵심! 부반송파 분산
     │
[부반송파 매핑] ← OFDM과 동일
     │
  [IFFT]
     │
[송신]
```

DFT로 데이터를 미리 **주파수 영역에 분산**시켜 놓으면
→ 부반송파들이 한꺼번에 피크로 겹칠 확률 급감
→ PAPR 대폭 감소 ✓

그래서 SC-FDMA를 **"DFT-spread OFDM" (DFT-s-OFDM)** 이라고도 부릅니다.

---

### 5G에서는?

5G NR에서도 동일한 원리 유지:
- 하향: **OFDM**
- 상향: **DFT-s-OFDM** (SC-FDMA의 5G 버전) — 기본값
- 단, 5G 상향은 조건에 따라 **CP-OFDM도 선택 가능** (단말 성능 좋을 때)

---

### 한 줄 요약

> 기지국은 전력 걱정 없으니 OFDM, 스마트폰은 배터리 아껴야 하니 PAPR 낮은 SC-FDMA로 설계
