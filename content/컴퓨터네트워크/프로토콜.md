SNMP: 간이 망 관리 프로토콜

ICMP (Internet Control Message Protocol): 네트워크 장치 간에 오류 메시지나 운영 정보를 주고받는 데 사용되는 프로토콜

IGMP (Internet Group Management Protocol): 호스트(PC)가 특정 멀티캐스트 그룹에 가입하거나 탈퇴함을 라우터에게 알리는 데 사용되는 프로토콜입니다.  라우터는 이를 통해 어떤 호스트에게 멀티캐스트 데이터를 보내야 할지 결정합니다

### 1. SNMP (Simple Network Management Protocol)

👉 네트워크 관리용 프로토콜

목적: 장비 상태 모니터링 + 제어

사용 예:<br>
CPU 사용률 확인<br>
트래픽 통계 수집<br>
장비 설정 변경<br>

구조:
Manager ↔ Agent

포트:<br>
UDP 161 (요청/응답)<br>
UDP 162 (Trap, 알림)

✔ 특징

사람이 아니라 관리 시스템이 쓰는 프로토콜
장비 내부 정보를 가져오는 용도

### 2. ICMP (Internet Control Message Protocol)

👉 에러 보고 + 상태 확인용

목적:
네트워크 오류 알림 연결 상태 확인

대표 예:

ping (Echo Request/Reply)<br>
traceroute<br>
포트: 없음 (IP 프로토콜 번호 1)

✔ 특징

데이터 전송용이 아니라 보조 제어용
라우터/호스트가 문제 상황 알려줌

### 3. IGMP (Internet Group Management Protocol)

👉 멀티캐스트 그룹 관리용

목적:
"이 멀티캐스트 그룹 받을래?" 가입/탈퇴

사용 예:  
IPTV  
실시간 방송 스트리밍  
포트: 없음 (IP 프로토콜 번호 2)  

✔ 특징

호스트 ↔ 라우터 사이에서만 동작\
멀티캐스트 트래픽 제어
