## 1. 거리 벡터 (Distance Vector)

대표: RIP, IGRP

### 동작 방식

* 각 라우터는 **목적지까지의 거리(metric)**와 **다음 홉(next hop)**만 안다.
* 이 정보를 **이웃 라우터에게 주기적으로 통째로 전달**한다.
* 알고리즘: **Bellman-Ford**

### 특징

* 구조 단순, 설정 쉬움
* 전체 네트워크 구조는 모름 (부분 정보만 있음)
* 수렴 속도 느림
* 루프 발생 가능 (count-to-infinity 문제)

### 요약

> “나는 어디까지 몇 홉인지 정도만 알고, 이웃이 알려준 정보 믿고 업데이트한다.”

---

## 2. 링크 상태 (Link State)

대표: OSPF, IS-IS

### 동작 방식

* 각 라우터는 **자신과 직접 연결된 링크 상태 정보를 전체 네트워크에 전파**
* 모든 라우터가 **동일한 네트워크 토폴로지 DB**를 가짐
* 알고리즘: **Dijkstra**

### 특징

* 네트워크 전체 구조를 정확히 앎
* 수렴 빠름
* 루프 거의 없음
* CPU/메모리 사용량 높음
* 설정 복잡

### 요약

> “나는 전체 지도 다 들고 있고, 거기서 최단경로를 직접 계산한다.”

---

## 핵심 비교

| 구분    | 거리 벡터        | 링크 상태    |
| ----- | ------------ | -------- |
| 정보 범위 | 이웃 기준        | 전체 네트워크  |
| 알고리즘  | Bellman-Ford | Dijkstra |
| 수렴 속도 | 느림           | 빠름       |
| 루프    | 발생 가능        | 거의 없음    |
| 자원 사용 | 적음           | 많음       |
| 확장성   | 낮음           | 높음       |

---

## 직관적인 비유

* 거리 벡터
  → “친구들한테 길 물어보면서 가는 방식”
  (정보 부정확, 느림)

* 링크 상태
  → “구글 지도 보고 최단경로 계산”
  (정확, 빠름)

---

## 한 줄 정리

* 거리 벡터 = **분산된 단편 정보 기반**
* 링크 상태 = **전체 지도 기반 계산**

### 거리벡터방식

## AS 기준 라우팅 분류 (IGP vs EGP)

네트워크를 관리하는 독립적인 영역인 AS(Autonomous System)를 기준으로 다음과 같이 나뉩니다.

### **IGP (Interior Gateway Protocol):** 동일한 AS 내부에서 라우팅 정보를 교환하는 프로토콜입니다.
    * **RIP (Routing Information Protocol):** 거리 벡터(Distance Vector) 알고리즘을 사용하는 대표적인 IGP입니다.
    * **OSPF (Open Shortest Path First):** 링크 상태(Link State) 알고리즘을 사용하는 대규모 네트워크용 IGP입니다.
    * IGRP / EIGRP: Cisco에서 개발한 거리 벡터 방식의 IGP입니다.


### **EGP (Exterior Gateway Protocol):** 서로 다른 AS 간에 라우팅 정보를 교환하는 프로토콜입니다.
    * **BGP (Border Gateway Protocol):** 현재 인터넷에서 AS 간 경로 설정을 위해 사용하는 유일한 표준 EGP입니다.

BGP는 신뢰성 있는 통신을 위해 TCP(포트 179) 연결을 맺고 동작합니다.  

반면 RIP는 UDP를 사용하며, 

OSPF는 IP 프로토콜을 직접 사용합니다

## 스플릿 호라이즌
라우터가 특정 인터페이스를 통해 네트워크 경로 정보를 수신했을 때, 그 정보를 다시 해당 인터페이스로 송신하지 않는 원리입니다.

목적: 잘못된 경로 정보가 네트워크 내에서 무한히 반복되어 도는 라우팅 루프(Routing Loop) 현상을 방지합니다. 

작동 방식: A 라우터가 B 라우터로부터 "네트워크 X는 나를 거치면 된다"는 정보를 받았다면, A는 B에게 다시 "네트워크 X는 나를 거치면 된다"고 알려줄 필요가 없습니다. 오히려 그렇게 하면 루프가 발생할 수 있기 때문에 이를 차단하는 것입니다.

최대 홉 카운트(Maximum Hop Count):  거리 벡터 알고리즘(RIP 등)에서 홉 카운트의 최대치(예: 15)를 설정하고, 이를 넘어가면(16) 도달 불가능한 경로로 간주하여 무한 루프를 끊습니다.


홀드 다운 타이머(Hold Down Timer):  특정 경로가 불량하다는 정보를 받으면, 해당 경로의 상태 변화를 일정 시간 동안 유보하여 전체 네트워크가 안정될 때까지 기다리는 방식입니다.


라우트 포이즈닝(Route Poisoning):  특정 경로가 다운되었을 때 해당 경로의 홉 카운트를 즉시 '무한대'로 설정하여 모든 라우터에 알리는 방식입니다

## BGP
BGP는 거리 벡터도, 링크 상태도 아니다. 정확히는 **Path Vector** 계열이다. 핵심은 “거리”가 아니라 **경로 자체(AS 경로)를 들고 다니면서 정책 기반으로 선택한다**는 점이다.

대표: BGP

* 인터넷(AS 간) 라우팅 프로토콜
* 목적: **최단경로가 아니라 정책 기반 경로 선택**
* 단위: 라우터가 아니라 **AS (Autonomous System)**

---

## 2. Path Vector가 뭐냐

### 개념

* 각 라우터는 단순히 “거리”가 아니라
  **목적지까지 지나가는 AS 목록(AS_PATH)**을 통째로 들고 다님

예:

```
목적지: 8.8.8.0/24
AS_PATH: 65001 → 65002 → 15169
```

### 의미

* “몇 홉”이 아니라
  → “어떤 경로를 거쳐가는지”가 핵심 정보

---

## 3. 왜 Distance Vector랑 다른가

거리 벡터 (예: RIP)는:

* metric = 홉 수
* 이웃이 준 값 누적
* 경로 자체는 모름

BGP는:

* metric 대신 **AS_PATH 자체**
* 루프 방지: **자기 AS 번호가 경로에 있으면 폐기**

👉 핵심 차이

> 거리 벡터: “몇 칸 떨어져 있음”
> BGP: “이 길로 가면 이런 경로를 지나감”

---

## 4. 왜 Link State랑 다른가

링크 상태 (예: OSPF)는:

* 전체 네트워크 토폴로지 DB 보유
* Dijkstra로 최단경로 계산

BGP는:

* **전체 인터넷 구조를 절대 모름**
* 이웃이 광고한 경로만 보고 선택

👉 핵심 차이

> 링크 상태: “전체 지도 기반 계산”
> BGP: “받은 경로 중 정책으로 선택”

---

## 5. BGP의 핵심 선택 기준 (여기서 본질 드러남)

BGP는 최단경로 안 쓴다. 대신:

1. Local Preference (정책)
2. AS_PATH 길이 (짧을수록 선호)
3. Origin
4. MED
5. eBGP > iBGP
6. IGP cost

👉 즉,

> “네트워크 운영자의 정책”이 최우선

---

## 6. 왜 Path Vector가 필요한가

인터넷은:

* 서로 다른 조직(통신사, 클라우드 등)이 연결된 구조

그래서:

* 단순 최단경로 쓰면 안 됨
* 특정 AS 피하거나, 특정 경로만 쓰는 정책 필요

예:

* A 통신사는 B는 쓰고 C는 안 쓰겠다
* 비용, 계약, 보안 이유

👉 이걸 표현하려면

* “거리”가 아니라 **“경로 자체”**가 필요

---

## 7. 루프 방지 방식 차이

* RIP: count-to-infinity → 문제 많음
* OSPF: 토폴로지 기반 → 구조적으로 방지
* BGP:
  → **AS_PATH에 자기 AS 있으면 바로 드랍**

이게 Path Vector의 강력한 특징

---

## 한 줄 핵심 정리

* Distance Vector = 거리 기반 (단순, 불완전)
* Link State = 전체 지도 기반 (정확, 빠름)
* BGP(Path Vector) = **경로 + 정책 기반 (인터넷용)**

---

## 감각적으로 이해

* RIP → “몇 정거장 떨어짐”
* OSPF → “지도 보고 최단거리 계산”
* BGP → “이 길은 KT→SK→Google 거쳐감, 이건 LG→AWS 거쳐감 → 정책으로 선택”
