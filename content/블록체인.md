---
title: 블록체인
---
블록에는 여러 개의 거래 내역이 있고, 이 거래들을 해시해서 하나의 머클루트를 만든다.

그리고 블록 헤더에는 이전 블록의 해시값과 이 머클루트가 포함된다.

이렇게 해서 블록들이 서로 이전 해시로 연결되어 체인이 되고, 이 데이터는 여러 노드에 복사되어 저장된다.

#### 블록은 크게 2개로 나뉨:

```
블록 헤더
트랜잭션 데이터
```

#### 블록 헤더 안에 들어가는 것:
이전 블록 해시 (Prev Block Hash)  
머클루트 (Merkle Root)  
타임스탬프 등

```{filename="블록 구조"}
[블록 헤더]
 - 이전 블록 해시  ← 체인 연결용
 - 머클루트       ← 현재 블록 데이터 요약
 - 타임스탬프     ← 블록 생성 시간

[트랜잭션 데이터]
 - Tx1
 - Tx2
 - Tx3
```

#### PoS(Proof of Stake)
스테이킹한 코인 양에 비례해 블록 생성자로 선택될 확률이 높아지는 합의 알고리즘. 에너지 효율적이며, 잘못된 행동 시 자산이 슬래싱된다.

  → 공격 시 자기 자산 날림 (슬래싱)
  → 경제적으로 공격 억제


#### PoW(Proof of Work)
해시 연산 퍼즐을 가장 먼저 푸는 노드가 블록 생성 권한을 얻는 알고리즘. 전력 소모는 크지만, 물리적 비용 기반 보안을 가진다.

  → 공격하려면 막대한 전력 + 장비 필요


| 구분    | PoW          | PoS           |
| ----- | ------------ | ------------- |
| 기준    | 연산 능력 (해시파워) | 스테이킹된 지분      |
| 비용    | 전기 + 장비      | 기회비용 (코인 락업)  |
| 보안 방식 | 물리적 비용 (전력)  | 경제적 패널티 (슬래싱) |


> PoW는 “연산 경쟁”, PoS는 “지분 기반 선택”
