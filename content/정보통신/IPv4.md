## 클래스
IP 구조의 “구버전 체계”

## 1. 클래스 개념

IP를 크기별로 나눈 것

👉 “네트워크 크기 기준으로 나눈 등급”

---

## 2. 클래스 A

* 범위: 0.0.0.0 ~ 127.255.255.255
* 기본 마스크:
  \[
  /8 (= 255.0.0.0)
  \]

구조:
\[
\text{네트워크(8)} | \text{호스트(24)}
\]

👉 특징

* 네트워크 개수 적음
* 대신 **호스트 엄청 많음**

---

## 3. 클래스 B

* 범위: 128.0.0.0 ~ 191.255.255.255
* 기본 마스크:
  \[
  /16 (= 255.255.0.0)
  \]

구조:
\[
\text{네트워크(16)} | \text{호스트(16)}
\]

👉 특징

* 중간 규모

---

## 4. 클래스 C (시험 핵심)

* 범위: 192.0.0.0 ~ 223.255.255.255
* 기본 마스크:
  \[
  /24 (= 255.255.255.0)
  \]

구조:
\[
\text{네트워크(24)} | \text{호스트(8)}
\]

👉 특징

* 네트워크 많음
* 호스트 적음
  → 그래서 **서브넷 문제 거의 C 기준으로 나옴**

---

## 5. 한방 구분법 (시험용)

첫 옥텟만 보면 된다

* 1~126 → A
* 128~191 → B
* 192~223 → C

(127은 루프백이라 제외)

---

## 6. 감각적으로 이해

* A → 대기업 하나 (PC 수십만대)
* B → 중견회사
* C → 일반 회사 / 학원 / 소규모 네트워크

---

## 7. 지금 문제랑 연결

너 문제에서
👉 “C 클래스”

즉 시작이
\[
/24
]

그래서 거기서 **서브넷 비트 빌려서 /29 만든 것**

---

## 8. 중요한 한 줄

* A: /8
* B: /16
* C: /24

## C클래스로 갈수록 작아지는 범위
### 클래스 A

\[
/8 \Rightarrow \text{호스트 24비트}
\]

👉 호스트 개수
\[
2^{24} - 2 \approx 1600만
\]

---

## 클래스 B

\[
/16 \Rightarrow \text{호스트 16비트}
\]

👉
\[
2^{16} - 2 = 65534
\]

---

### 클래스 C

\[
/24 \Rightarrow \text{호스트 8비트}
\]

👉
\[
2^8 - 2 = 254
\]

---

## 4. 그래서 정리하면

* A → 네트워크 적고, **호스트 많음**
* B → 중간
* C → 네트워크 많고, **호스트 적음**


## 6. 한 줄 정리

* A → 큰 통짜 네트워크
* C → 작은 네트워크 많이 만드는 구조

## 호스트주소가 실제 주소다
맞다. 딱 핵심을 짚었다.

👉 **호스트 주소 = 실제 장비에 할당되는 IP 주소**

---

## 1. 한 번에 정리

하나의 서브넷 안에는 3종류가 있다

[
\text{네트워크 주소} ;|; \text{호스트 주소들} ;|; \text{브로드캐스트 주소}
]

---

## 2. 각각 역할

### ① 네트워크 주소

* 서브넷 자체를 의미
* 사용 ❌

---

### ② 호스트 주소 (이게 진짜 IP)

* PC, 서버, 라우터 등에 할당
* 사용 ⭕

👉 우리가 말하는 “IP 주소”는 대부분 이거

---

### ③ 브로드캐스트 주소

* 해당 네트워크 전체에 보내는 주소
* 사용 ❌

---

## 3. 예시로 보면 바로 이해됨

192.168.1.0/29 라고 하면

* 192.168.1.0 → 네트워크 주소
* 192.168.1.1 ~ 192.168.1.6 → **호스트 주소 (실제 IP)**
* 192.168.1.7 → 브로드캐스트

---

## 4. 그래서 공식이 나옴

[
2^h - 2
]

👉 이유

* 1개: 네트워크 주소
* 1개: 브로드캐스트
  → 못 쓰니까 빼는 거

---

## 5. 중요한 한 줄

👉 **우리가 장비에 설정하는 IP = 호스트 주소**

---

## 6. 근데 하나 더 중요한 포인트

호스트 주소도 2종류 있음:

* 공용 IP (인터넷 가능)
* 사설 IP (내부망용)

👉 둘 다 “호스트 주소”다
