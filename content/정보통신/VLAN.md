## 1. VLAN이란?

VLAN (Virtual LAN)은  
**하나의 물리적 네트워크를 여러 개의 논리적 네트워크로 나누는 기술**이다.

- 같은 스위치에 연결되어 있어도 VLAN이 다르면 서로 통신 불가
- 브로드캐스트 도메인을 분리하는 효과

---

## 2. VLAN을 사용하는 이유

- 부서별 네트워크 분리 (개발 / 회계 / 운영)
- 보안 강화 (다른 VLAN 간 접근 차단)
- 브로드캐스트 트래픽 감소
- 네트워크 관리 효율 향상

---

## 3. VLAN 핵심 개념

### (1) VLAN ID
각 VLAN은 고유 번호로 구분된다.

예:
- VLAN 10
- VLAN 20
```
[PC1] \
[PC2]  >--- [소형 스위치] --- (스위치 1번 포트)
[PC3] /
```
---

### (2) 브로드캐스트 도메인 분리

- 같은 VLAN → 통신 가능
- 다른 VLAN → 통신 불가

---

### (3) 포트 기반 VLAN

스위치 포트를 기준으로 VLAN을 할당한다.

예:
- 포트 1 → VLAN 10
- 포트 2 → VLAN 20

---

## 4. Access Port vs Trunk Port

### Access Port
- 하나의 VLAN만 속함
- PC 같은 단말 장비 연결

---

### Trunk Port
- 여러 VLAN을 하나의 링크로 전달
- 스위치 간 연결에 사용
- VLAN 태깅 방식 사용 (IEEE 802.1Q)
```
[PC1] --(VLAN 10)--\
                    [Switch] ---- (Trunk) ---- [Switch]
[PC2] --(VLAN 20)--/
```
---

## 5. PC 여러 대를 한 포트에 연결할 수 있을까?

### 결론
👉 직접 연결은 불가능  
👉 중간에 스위치를 사용해야 함

## 6 VLAN 간 통신 (Inter-VLAN)

VLAN 간 통신은 기본적으로 불가능하다.

통신하려면:
- L3 스위치 또는 라우터 필요


## 트렁킹
# VLAN 트렁킹(Trunking) 정리

## 1. 트렁킹이란?

트렁킹(Trunking)은  
**하나의 물리적 링크로 여러 VLAN의 트래픽을 동시에 전달하는 기술**이다.

👉 쉽게 말하면  
**“한 선으로 여러 네트워크(VLAN)를 같이 보내는 것”**

---

## 2. 왜 필요한가?

스위치가 여러 대 있을 때를 생각해보면:

### 트렁킹 없을 때
- VLAN 10용 케이블 1개
- VLAN 20용 케이블 1개
- VLAN 30용 케이블 1개

👉 VLAN마다 선이 따로 필요 → 비효율

---

### 트렁킹 사용 시
- 케이블 1개로 모든 VLAN 전달 가능

👉 훨씬 효율적

---

## 3. 동작 원리 (핵심)

트렁킹은 **프레임에 VLAN 태그를 붙여서 구분**한다.

### VLAN 태깅 (Tagging)

- 송신 시: VLAN ID를 프레임에 추가
- 수신 시: VLAN ID 보고 어떤 VLAN인지 판단

---

### 4. 표준: IEEE 802.1Q

트렁킹에서 사용하는 표준

### 특징
- 이더넷 프레임에 VLAN 태그 추가
- VLAN ID 포함

---

### 5. 구조 예시
```
[Switch A] ===(Trunk Port)=== [Switch B]
```


이 링크 하나로:

- VLAN 10
- VLAN 20
- VLAN 30

모두 전달 가능

---

### 6. Access vs Trunk 비교

| 구분 | Access Port | Trunk Port |
|------|------------|------------|
| VLAN 수 | 1개 | 여러 개 |
| 대상 | PC | 스위치, 서버 |
| 태깅 | 없음 | 있음 (802.1Q) |

---

### 7. Native VLAN

트렁크에서 **태그 없이 보내는 VLAN**

- 기본 VLAN (보통 VLAN 1)
- 태그 없이 전송됨

👉 주의: 보안 문제 때문에 변경하는 경우 많음

---

### 8. 실제 흐름

예를 들어:
```
PC → Switch A → (Trunk) → Switch B → PC
```

1. PC가 프레임 전송 (태그 없음)
2. Switch A가 VLAN 10 태그 추가
3. Trunk 링크로 전달
4. Switch B가 태그 제거 후 전달

---

### 9. 언제 사용하냐

- 스위치 ↔ 스위치 연결
- 스위치 ↔ 서버 (가상화, 여러 VLAN 필요)
- 스위치 ↔ 라우터 (Router-on-a-Stick)

---

### 10. 핵심 요약

- 트렁킹 = 한 포트로 여러 VLAN 전달
- 802.1Q 태깅 사용
- 스위치 간 연결 필수 기술
- VLAN 확장할 때 핵심 역할

## VTP(VLAN Trunking Protocol)의 3가지 운용 모드

서버 모드 (Server Mode): VLAN의 생성, 수정, 삭제가 가능하며, 자신의 VLAN 정보를 다른 스위치들에게 광고하여 동기화를 주도하는 기본 모드입니다. 


클라이언트 모드 (Client Mode): 스스로 VLAN을 생성하거나 수정할 수 없으며, 서버로부터 받은 정보를 통해서만 자신의 VLAN 정보를 업데이트(동기화)합니다. 

트랜스페이런트 모드 (Transparent Mode): 외부에서 받은 VTP 정보를 자신에게 적용(동기화)하지 않고 단순히 전달(중계)만 합니다. 하지만 자기 자신을 위한 VLAN의 생성, 수정, 삭제는 독립적으로 가능합니다.
